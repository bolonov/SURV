Добрый день!

 

> 1 ) правильно ли мы понимаем , что основная таблица событий И таблицы

> настроек системы находятся в разных БД ? ( события в

> GGLite_01 , а все настройки в GGLite )

 

Конфигурация системы хранится в базе данных GGLite , а журнал событий сначала сохраняется в базе данных GGLite_01 , а затем может храниться в базах данных GGLite_nn с другим числовым окончанием.

 

Если в окне " Настройка " на вкладке " Параметры журналов событий "

включен режим проверки размера журнала событий , то после достижения базой данных GGLite_01 максимального размера будет создана база данных GGLite_02 , после чего последующие события будут храниться в базе данных GGLite_02 , и т. д.

 

Такая организация данных связана с ограничением SQL - сервера " Microsoft SQL Server 2000 Desktop Engine ( MSDE ) " , который прекращал работу с базой данных после увеличения ее размера до 2 ГБ .

 

> 2 ) можем ли создать в одной из ЭТИХ баз Представление ( View ) ? ( в

> частности НЕ повлияет ли это каким-то образом на Лицензионное

> соглашение ...

 

Использование SQL - сервера для работы с данными ПО " Golden Gate 2002" не является нарушением лицензионного соглашения с компанией Microsoft .

 

> ... и работоспособность системы ?)

 

При одновременной работе с базами данных ПО " Golden Gate 2002"

стороннего программного обеспечения мы не можем гарантировать работоспособность ПО " Golden Gate 2002" .

 

Запрос SELECT , в котором используется View или таблица базы данных , может негативно повлиять на работу ПО " Golden Gate 2002" , например , следующим образом:

 

1 ) блокирование работы ПО " Golden Gate 2002" во время модификации базы

   данных , если в это время стороннее программное обеспечение выполняет запрос

   в пределах транзакции с уровнем изоляции REPEATABLE READ или

   SERIALIZABLE .

   

2 ) Одновременное выполнение большого количества запросов может существенно

   замедлить работу SQL - сервера , особенно в случае использования

   " Microsoft SQL Server 2000 Desktop Engine ( MSDE ) " .

 

> 3 ) если создать свою View -ху допустим , это не пропадет ли Она из

> структуры БД при следующем обновлении системы GoldenGate ?

> ( И вообще каким образом обычно это обновление происходит ?)

 

При запуске программы обновления базы данных осуществляется проверка номера версии базы данных , наличия всех таблиц , полей и индексов .

Отсутствуют таблицы , поля и индексы создаются автоматически.

 

Сейчас обновления системы не влияет на посторонние объекты в базах данных GGLite , GGLite_01 , GGLite_02 и других базах данных ПО " Golden Gate 2002" , но в будущем мы не можем исключать возможность такого влияния .

 

С целью исключения влияния ПО " Golden Gate 2002" на посторонние объекты Вы можете создать новую базу данных для своего программного обеспечения.

В этой базе данных можно создать view с использованием данных из баз GGLite , GGLite_01 и других баз данных ПО " Golden Gate 2002" .
